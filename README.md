# 选股策略回测系统

基于 [thsdk](https://pypi.org/project/thsdk/)（同花顺数据接口）的选股策略回测工具，BS 架构（浏览器 + 服务端）。

- 用 `wencai_nlp`（问财自然语言）按策略选股，支持历史日期回测
- 以**上证指数 K 线**作为交易日历，天然排除周末/节假日/临时休市，自动推算 T-1/T-2/T-3 与 T+1/T+2/T+3
- 结果表展示选股日股价及后 1/2/3 个交易日的累计涨跌幅（涨红跌绿）
- 复刻通达信「拉升资金」指标，对**连续 3 天拉升资金增加**的股票标注红色「主」徽标（悬停显示提示）
- 流式回测带**进度条**，并可展开查看每日的 wencai 请求与原始返回，便于排查

## 技术栈

- Python ≥ 3.10，依赖管理用 [uv](https://docs.astral.sh/uv/)
- 后端：FastAPI + Uvicorn
- 数据：thsdk、pandas

## 文件结构

| 文件 | 说明 |
| --- | --- |
| `app.py` | FastAPI 后端：回测 REST/流式接口、策略库 CRUD 接口，托管前端 |
| `core.py` | 回测核心：THS 连接、交易日历、选股、行情、拉升资金指标、{T} 本地子句核验 |

| `static/index.html` | 前端单页：表单、策略选择/保存、进度条、结果表、调试面板 |
| `config.example.json` | 账户配置模板（复制为 `config.json` 使用） |
| `pyproject.toml` / `uv.lock` | 依赖声明与锁定 |

## 在 Windows 计算机上运行

### 一、需要拷贝的文件

整个项目目录，但**排除**以下内容（不可移植或含敏感信息）：

- `.venv/` —— 虚拟环境含平台相关二进制，必须在目标机重建
- `config.json` —— 含账户凭证，到新机器重新配置
- `__pycache__/`

需保留：`app.py`、`core.py`、`config.py`、`static/`、`pyproject.toml`、`uv.lock`、`config.example.json`、`.python-version`。

### 二、安装 uv

无需单独安装 Python，uv 会按 `.python-version` 自动下载 3.10。在 PowerShell 执行：

```powershell
powershell -ExecutionPolicy ByPass -c "irm https://astral.sh/uv/install.ps1 | iex"
```

安装后**重开一个终端**使 PATH 生效，验证：

```powershell
uv --version
```

### 三、安装依赖

进入项目目录，按 `uv.lock` 还原环境（自动下载 Python 与全部依赖）：

```powershell
cd 项目路径
uv sync
```

> `uvicorn[standard]` 中的 `uvloop` 仅在 Linux/Mac 安装，Windows 自动跳过并回退到 asyncio，不影响运行。thsdk 为纯 Python 包，无需编译器。

### 四、配置账户（可选）

不配置则使用临时游客账户（可测试，但可能随时失效）。使用自己的同花顺账户：

```powershell
copy config.example.json config.json
```

编辑 `config.json` 填入 `username` / `password`（`mac` 可留空）。也可用环境变量临时覆盖：

```powershell
$env:THS_USERNAME="你的账号"; $env:THS_PASSWORD="你的密码"
```

### 五、启动

```powershell
uv run uvicorn app:app --host 0.0.0.0 --port 8000
```

浏览器打开 **http://localhost:8000**，填写回测日期区间后点「开始回测」。

### 注意事项

- **网络**：thsdk 需连接同花顺行情服务器，确保能联外网；首次运行若 Windows 防火墙弹窗，允许 Python 通过。
- **访问范围**：`--host 127.0.0.1` 仅本机访问；`--host 0.0.0.0` 则同局域网设备可经 `http://本机IP:8000` 访问。
- **端口冲突**：8000 被占用时改用 `--port 8001`。
- **离线内网机器**：目标机无法联公网安装依赖时，可在有网机器 `uv export > requirements.txt` 或 `uv pip download` 预下载后拷入内网；但 thsdk 运行时仍需能连同花顺服务器。

## 策略说明

默认策略模板在 `core.py` 的 `DEFAULT_STRATEGY_TEMPLATE`，前端可临时编辑覆盖。日期占位符：

- `{T}` —— 选股日
- `{T1}` / `{T2}` / `{T3}` —— T 往前推的第 1 / 2 / 3 个交易日

运行时占位符替换为「YYYY年M月D日」交给 wencai 识别。

### 策略库（多策略保存与选择）

策略以「名字 → 模板」键值对存于本地 SQLite（`strategies.db`，已 gitignore，不含凭证）。前端策略栏：

- **下拉选择**：选中后把该策略模板填入文本框；**文本框是回测的唯一事实来源**（真正发送的以它为准）。
- **＋ 新增**：输入名字，以当前文本框内容建一条新策略。
- **💾 保存**：把当前文本框内容存为指定名字（同名覆盖、新名新增，upsert）。
- **🗑 删除**：删除当前策略（默认策略受保护，不可删除）。

接口：`GET/POST /api/strategies`、`DELETE /api/strategies/{name}`；存储实现见 `strategies.py`。

### {T} 本地核验子句（不送问财，用 9:26 开盘价自算）

凡涉及 `{T}` 当日实时状态的指标都不能交问财（问财用 {T} 收盘价，含未来信息且非 9:26 状态）。
引擎在送问财前**自动剥离**下列子句并本地核验，可写进任意策略，写不写、写哪个都行：

| 子句写法 | 含义 | 核验 |
| --- | --- | --- |
| `{T}均线角度大于N` | 阈值型，`角度 > N` | `_ma5_angle`（9:26 开盘价） |
| `{T}均线角度大于{Tk}前均线角度` | 比较型，`{T}角度 > 第 k 个交易日前角度`（k=1/2/3） | `_ma5_angle` 双算对比 |
| `{T}6日均线大于中轨线` | `MA(C,6) > 中轨线` | `_midline_ok` |

中轨线（通达信自定义指标，问财不认）：`压力线=REF(HHV(H,N),1)`、`支撑线=REF(LLV(L,N),1)`、`中轨线=(压力线+支撑线)/1.9`，参数 `N=10`（`core.py` 的 `_MIDLINE_N` / `_MIDLINE_DIVISOR` / `_MA_MID`）。`{T}` 的 6 日均线同样用 9:26 开盘价替换今日收盘价。
`{T1}/{T2}/{T3}` 的同类子句已收盘、与问财一致，继续交问财判定。

## 5 日均线角度：公式反推与 9:26 自算

### 结论（已实测标定）

问财的「5 日均线角度」并非数据固有量，而是把均线斜率换算成角度时人为选定的「坐标尺子」。
用 30 只跨价位股票在已收盘交易日反推（脚本见下），结论是一个精确解析式：

```
角度 = arctan( K × (MA5今 / MA5昨 − 1) )          K = 100
     = arctan( 5 日均线单日涨幅的百分数值 )
```

- **归一口径 = 涨幅**（不是绝对元价差）：绝对元口径拟合残差约 25°，被证伪。
- **复权口径 = 前复权**（与问财一致）。
- **系数 = 干净整数 100**：5 日均线单日涨 **1% ⟺ 恰好 45°**；阈值含义示例 `>65°` ⟺ 5MA 单日涨幅 > tan65° ≈ **2.14%**。
- 30 只样本回算 RMSE ≈ 0.0004°，逐股完全吻合。

标定/验证脚本：`calibrate_ma5_angle.py`（四口径判优）、`refine_k.py`（精修锁定 K=100）、`probe_wencai.py`（接口探针）。

### 为什么 {T} 角度要在 9:26 自算

问财的均线角度用 **{T} 收盘价**，这对一个「9:26 竞价撮合后」的选股决策既**含未来信息**、又**非 9:26 当下状态**。
因此引擎把 {T} 角度从问财查询中剥离，改为用上式在本地计算：以 **{T} 开盘价**（集合竞价 9:25 已定价，等同 9:26 价）替换今日收盘价，得到
`MA5今 =（前 4 日收盘 + 今日开盘）/ 5`，再套同一公式。
`{T1}/{T3}` 角度仍交问财判定（已收盘交易日，本公式与问财完全一致，无需改动）。

**阈值以模板为唯一事实来源**：模板里的「{T}均线角度大于N」由引擎运行时解析出 N，并在送问财前剥离该子句，改用 9:26 开盘价本地核验 `角度 > N`。改阈值只改模板即可，无需动代码。
相关实现在 `core.py`：公式常量 `_ANGLE_K=100`、`_ANGLE_ADJUST="forward"`；本地子句解析/剥离 `_parse_local_checks` / `_strip_local_clauses`；逐股核验 `_pass_local_checks`；角度与窗口计算 `_ma5_angle` / `_fwd_window`（详见上文「{T} 本地核验子句」表）。

### 角度为何用前复权（与买卖价的不复权各司其职）

角度只取决于均线的**比值** `MA今/MA昨`，与价格绝对大小无关，由此：

- **窗口之后的分红不影响角度**：前复权会把窗口内 6 天价格统一缩放同一系数，分子分母同乘约掉，比值不变——所以前复权「用到未来分红」在角度上**不构成未来函数**。
- **窗口之内的分红必须前复权**：除权当天价格有机械跳空，不复权会把这个假缺口算进均线、角度失真；前复权抹平后才是真实斜率，而**这正是问财 9:26 当时显示的值**（标定时即据此确认问财用前复权、K=100）。

因此 **角度用前复权 = 精确还原问财 9:26 的角度**；而**买入价与后续涨幅用不复权**（真实可成交价）。两套口径各为其用，互不影响。

### 买入价 = {T} 开盘价

选股发生在 9:26，故回测的买入基准价（`base_price`）取 **{T} 开盘价**（不复权，真实可成交），
后 1/2/3 个交易日的涨跌幅均相对该开盘价计算。

## 指标「当天变化大」性质与口径取舍

混合方案的原则：**客观且当天不变**的数据走问财；**当天会随盘中变化**、问财又只能给收盘口径的，需自算或标注。
对默认模板逐条核查：

| 指标 | 盘中是否变化 | 处理 |
| --- | --- | --- |
| 剔除 ST、主板/创业板 | 否（名称/代码） | 问财 |
| 净利润同期增长、机构数 | 否（基本面） | 问财 |
| 竞价异动 / 竞价金额 / 竞价涨幅 / 高开 / 竞价急速上涨等 | 否（9:25 集合竞价后即定型） | 问财（与 9:26 一致） |
| {T1}/{T2}/{T3} 均线角度、{T2}/{T1} 开收盘关系 | 否（已收盘交易日） | 问财 |
| **{T} 均线角度** | **是（随价格变）** | **已改为 9:26 开盘价本地自算**（见上） |
| **{T} 所属板块涨幅** | **是（盘中剧烈变化）** | 见下方说明 |
| 流通市值（60 亿~500 亿） | 轻微（= 流通股 × 价格，流通股不变） | 暂留问财（区间宽，风险低） |

### {T} 所属板块涨幅：为何回测暂不替代

板块涨幅盘中变化剧烈，问财给的是**收盘口径**，与 9:26 竞价后的板块状态不同。理论上可用 thsdk 拼出来
（`ths_industry`/`ths_concept`/`block_constituents` 取板块归属 + `market_data_block` 取板块行情），
**但仅适用于实盘 9:26 的实时取值**：历史回测拿不到「过去某日 9:26 的板块快照」，
`market_data_block` 是实时数据、`klines` 也无板块的集合竞价分钟线。
因此**回测**中 {T} 板块涨幅继续走问财（收盘口径，属已知偏差）；**实盘**落地时应改为
「板块归属 + `market_data_block` 实时涨幅」在 9:26 自取。
